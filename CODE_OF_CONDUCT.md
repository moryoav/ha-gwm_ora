# Code of Conduct

## Our Pledge

We want GWM ORA for Home Assistant to be a welcoming project for people with different backgrounds, skill levels, vehicles, regions, and Home Assistant setups.

Contributors, maintainers, and users are expected to participate respectfully and constructively in issues, pull requests, discussions, and any other project spaces.

## Expected Behavior

Examples of behavior that helps this project:

- Be kind, patient, and constructive.
- Assume good intent when possible.
- Give feedback that is specific and useful.
- Respect different experience levels.
- Keep discussions focused on the project and the problem being solved.
- Be careful when discussing GWM credentials, vehicle location, security PINs, remote commands, logs, or personal Home Assistant configuration.

## Unacceptable Behavior

Examples of behavior that is not acceptable:

- Harassment, threats, insults, or personal attacks.
- Discriminatory language or behavior.
- Trolling, repeated disruption, or bad-faith arguments.
- Publishing private information without permission.
- Sharing another person's credentials, tokens, security PINs, VINs, precise vehicle location, logs, or private Home Assistant configuration.
- Encouraging unsafe remote vehicle control without clearly explaining the risk.

## Enforcement

Maintainers may remove comments, close issues or pull requests, limit participation, or take other reasonable action when behavior harms the project or its community.

If you see behavior that violates this Code of Conduct, please report it to the project maintainer. If the issue involves a security vulnerability, follow `SECURITY.md` instead of posting exploit details publicly.

## Scope

This Code of Conduct applies in all project spaces, including GitHub issues, pull requests, discussions, and any community spaces that represent the project.
